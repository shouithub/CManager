help_outline 说明与注意事项

1. 切换后端不会自动迁移历史文件。切换到 S3 前，建议先停止写入新文件，手动迁移后再启用。

2. Office Online 预览依赖直链。S3 模式下，传给 Office Online embedding 的 URL 必须能被微软服务器公网匿名访问，因此：

  • Bucket ACL 必须允许匿名读取（或为对应文件设置公开读取）；
  • 建议配置 自定义域名 / CDN 加速预览，避免 S3 跨地域拉取慢；
  • 未配置 CDN 时使用 S3 默认 URL，需保证 endpoint 域名可公网解析。

3. Secret Access Key 明文存储。请确保数据库访问权限严格受控；建议在 S3 后端为该 AK/SK 配置最小权限策略（仅 bucket 读写）。

4. 切换后端立即生效，无需重启服务。所有 FileField 自动走新后端，但已存文件链接（如 /media/form_submissions/...）不会自动重写为 S3 直链。

5. 使用 S3 的核心收益：本站带宽仅 3 Mbps，大文件（合并 Word、PDF、图片）通过 S3 直链可以绕开本站带宽瓶颈，显著提升用户上传/下载/预览速度。
